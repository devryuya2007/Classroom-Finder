## 2. データモデル & スキーマ

### 2.1 データ型（JSDoc）
```js
/** @typedef {'assignment'|'material'|'post'|'comment'} DocType */
/** @typedef {'docs'|'sheets'|'slides'|'pdf'|'link'|'other'} AttachmentKind */

/** @typedef {{ kind: AttachmentKind, title: string, url: string }} Attachment */

/** @typedef {{
 *  id: string,
 *  url: string,
 *  type: DocType,
 *  className: string,
 *  teachers: string[],
 *  topic?: (string|null),
 *  title: string,
 *  text: string,
 *  due?: (number|null),
 *  attachments: Attachment[],
 *  updatedAt: number
 * }} Document */

````

### 2.2 IndexedDB

- DB 名：`gcx-search`
- バージョン：`1`（将来 `2+` に上げる場合は `meta` に移行履歴記録）
- ストア
  - `documents`（PK=`id`、索引：`className`, `type`, `due`, `updatedAt`）
  - `meta`（`version`, `lastCrawl`, `stats` 等）
- ガーベジポリシー
  - 抜粋 `text` は最大長（例：1,000 文字）でトリム
  - 最古データのローリング（上限件数/MB 超過時）

## 3. DOM 抽出設計

### 3.1 対象

- クラス一覧 → 各クラスの「ストリーム/課題/資料/コメント」
- 添付（Drive/外部リンク）の**タイトル/ファイル名**（本文は読まない）

### 3.2 セレクタ戦略（多層フォールバック）

1. **data-属性/ARIA/role** を最優先
2. ラベル近傍・見出し階層からの構造ヒューリスティック
3. 安定クラス名の組合せ（単独クラス名への依存を回避）
4. 複数候補を**配列で順次試行**し、成功率を `meta.stats` に記録

### 3.3 差分更新

- `MutationObserver`（childList/subtree）でカード追加/変更を検知
- 無限スクロールは `IntersectionObserver` で下部に到達時に追加走査
- レート制限：`throttle(1000ms)` / `requestIdleCallback` 併用
- アップサート条件：`id` 一致 →`updatedAt` 比較で更新判定

### 3.4 正規化

- `innerText` ベースでプレーンテキスト化
- 改行/空白の圧縮、制御文字除去
- 期日文字列 →epoch 変換（失敗時は `null`）

## 4. 検索設計

### 4.1 Fuse.js 設定（初期）

```js
/** @type {import('fuse.js').IFuseOptions<any>} */
const options = {
  keys: [
    { name: "title", weight: 0.35 },
    { name: "topic", weight: 0.2 },
    { name: "text", weight: 0.18 },
    { name: "className", weight: 0.12 },
    { name: "teachers", weight: 0.1 },
    { name: "attachments.title", weight: 0.05 },
  ],
  threshold: 0.3,
  ignoreLocation: true,
  includeScore: true,
  minMatchCharLength: 1,
};
```

### 4.2 日本語対策

- サジェスト用：先頭一致辞書（`title/topic/className/teachers` のユニーク一覧）
- あいまい一致：入力長が短い場合は bi/tri-gram を内部生成してクエリを補助
- 空白区切り=AND、`"..."`=フレーズ一致

### 4.3 フィルタ適用順

1. Fuse の候補（スコア順）を取得
2. **教師/期日/添付タイプ**の AND フィルタを適用
3. 上位 N 件（デフォ 20）を返す（スクロールで追加）

### 4.4 パフォーマンス方針

- インデックスはメモリ常駐、DB 更新時のみ再構築
- 入力イベントは `debounce(80–120ms)`＋`requestAnimationFrame`
- 目標：インデックス済 10k 件規模で **P95 ≤ 150ms**

## 5. UI/UX 設計

### 5.1 配置

- Classroom 標準サイドバー直下に**検索バー**（`position: sticky`）
- 画面上部のナビゲーションバー（`nav.joJglb[role="navigation"]`）右側に**クイック検索**（独自クラスで非干渉）
- 結果パネルはオーバーレイ（最大 20 件）

### 5.2 コンポーネント

- `Sidebar`：検索入力、フィルタチップ、設定アイコン
- `TopbarQuickSearch`：上部バー内の簡易検索入力（`gcx-topbar`, `gcx-topbar-input`）
- `ResultList`：タイトル/抜粋/クラス/タイプ/期日/添付アイコン、ヒットハイライト
- `Shortcuts`：`/` フォーカス、`Ctrl+K` コマンド、`↑↓Enter` 選択、`Esc` クローズ

### 5.3 アクセシビリティ

- `role="search" | "listbox" | "option"`, `aria-activedescendant`
- フォーカストラップ、`Esc`で閉じる
- テーマ追従（ダーク/ライト）、コントラスト基準を満たす

注：TopbarQuickSearch は Classroom のメニュー実装（ロービング tabindex/委譲クリック）と干渉しないよう、独自クラスとイベント伝播の遮断を併用する。

## 6. DOM ↔ API 切替

### 6.1 健全性メトリクス

- `selectorFailureRate`：抽出失敗/試行
- `indexCoverage`：想定件数に対する取得率
- `searchLatencyP95`：検索遅延（ms）

### 6.2 しきい値（初期）

- `selectorFailureRate > 15%` **or** `indexCoverage < 60%` **or** `P95 > 220ms（10k件超）`
  - ⇒ モーダルで**API モード提案**（OAuth 同意 → 取得 →DB 保存 →Fuse 再構築）

### 6.3 API モード（概要）

- Classroom API（読取スコープ）で courses / courseWork / materials / topics / teachers を取得
- 取得結果を **Document** スキーマに変換し IndexedDB に保存
- 保存後、Fuse 再構築 → UI は DOM 時と同一

---

## 7. セキュリティ / プライバシー / 権限

- 既定：**外部送信ゼロ**
- API モード時のみ：OAuth 通信＋取得（読取のみ）
- 描画はテキストノード中心（`innerHTML` 禁止）
- CSP：MV3 既定 + 追加スクリプト禁止
- 権限（最小）
  - `host_permissions`: `https://classroom.google.com/*`
  - `permissions`: `storage`, `scripting`
  - `background`: Service Worker
  - `content_scripts`: `run_at: document_idle`

---

## 8. 性能予算 & 計測

| 指標                   | 目標                        |
| ---------------------- | --------------------------- |
| 入力 → 候補表示（P95） | ≤ 150ms（インデックス済み） |
| 初回クロール           | Idle 分割で体感阻害なし     |
| 再インデックス         | 差分時のみ。フルは手動/Idle |

- 計測：`performance.now()`、統計は `meta.stats` に集約、`options` で可視化

---

## 9. エラーハンドリング

- 抽出 0 件/急減：トースト＋「再スキャン」「API に切替」ボタン
- API 失敗：指数バックオフ、トークン失効時は再同意
- DB 破損：DB バージョン+1 で再生成 → フル再クロール誘導
- 検索 0 件：サジェスト候補/近似語提示

// 差分抽出（content.js）
const scan = throttle(() => {
  const nodes = extractNodesWithFallback(document); // util/domExtract
  const docs = nodes.map(toDocumentModel); // 正規化/添付メタ
  db.upsertDocuments(docs);
  search.rebuildIfNeeded();
}, 1000);

new MutationObserver(() => requestIdleCallback(scan)).observe(document.body, {
  childList: true,
  subtree: true,
});

window.addEventListener("scroll", throttle(scan, 1500));

// 検索（search.js）
export function query(q: string, f: Filters) {
  const tokens = parse(q); // AND/"..."
  const results = fuse.search(tokens); // スコア順
  return applyFilters(results, f) // 教師/期日/添付
    .slice(0, 20)
    .map(decorateHighlight);
}

// 健全性チェック→API提案（bg.js）
setInterval(() => {
  const s = stats.read();
  if (s.failRate > 0.15 || s.coverage < 0.6 || s.p95 > 220) {
    promptApiMode(); // 同意→取得→保存→再構築
  }
}, 60_000);
